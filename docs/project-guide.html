PART 1: The Very Basics

What is this app actually doing?

Simple way to think about it:

Imagine you have a photograph.

Your photo is made of millions of tiny dots called pixels.

Each pixel has three colors mixed together:

  • Red (how much red)
  • Green (how much green)
  • Blue (how much blue)

This app takes these three colors in each pixel and changes them using math.

That's it.

The math part is like a recipe:

  • Old Red + Old Green + Old Blue → New Red
  • Old Red + Old Green + Old Blue → New Green
  • Old Red + Old Green + Old Blue → New Blue

The app shows you what changed, and also shows you the "recipe" (the math).

PART 2: Understanding X in R^(N×3)

The Image as a Big Table

What does R^(N×3) mean in super simple words?

Let me break this down letter by letter:

R = Real numbers (just regular numbers, like 1, 2, 3.5, etc.)

N = How many pixels you have

x = "by" or "times" (like 3 x 4 means 3 times 4)

3 = Always 3 (Red, Green, Blue)

So R^(N×3) means: "A table of real numbers with N rows and 3 columns."

Visual way to understand:

If your photo is 10 pixels wide and 10 pixels tall, you have 100 pixels total.

So N = 100.

Now make a table:

Row 1 (Pixel 1):     [ R=255,  G=100,  B=50  ]
Row 2 (Pixel 2):     [ R=200,  G=150,  B=75  ]
Row 3 (Pixel 3):     [ R=180,  G=180,  B=100 ]
...
Row 100 (Pixel 100): [ R=50,   G=50,   B=50  ]

This table is X in R^(100×3).

Each row = one pixel.
Each column = one color (Red, Green, or Blue).

Why do we do this?

Because with a table, we can use multiplication to change all pixels at once.

Instead of changing pixel 1, then pixel 2, then pixel 3... we can use one "recipe" and apply it to all 100 rows at once.

PART 3: Understanding v' = T v

The Magic Recipe

Simple analogy:

Imagine a recipe for lemonade:

Recipe (T):

  • Take 2 cups of lemon juice
  • Add 1 cup of sugar
  • Add 3 cups of water

Input (v):

  • 1 cup of lemon juice = [1, 0, 0]
  • 1 cup of sugar = [0, 1, 0]
  • 1 cup of water = [0, 0, 1]

Output (v'):

  • Using the recipe on [1, 0, 0] gives [2, 1, 3]

This is exactly what v' = T v means.

In color terms:

Input pixel v:

v = [100 Red, 50 Green, 30 Blue]

Recipe matrix T (a 3×3 table of numbers):

T = [
  [0.5,  0.2,  0.1],
  [0.3,  0.6,  0.2],
  [0.2,  0.2,  0.7]
]

New pixel v':

To get new Red:

  • Take old Red (100) and multiply by first number (0.5) → 50
  • Take old Green (50) and multiply by second number (0.2) → 10
  • Take old Blue (30) and multiply by third number (0.1) → 3
  • Add them: 50 + 10 + 3 = 63 → New Red = 63

Repeat for Green and Blue.

Result: v' = [63, 60, 44]

The pixel changed!

Why have a matrix (recipe)?

Because this one T can be used on all 100 pixels (or 1 million pixels) at once.

That's the power of linear algebra.

PART 4: Optional Operations After Transform

Stages After the Recipe

Timeline of what happens to your pixel:

Stage 1: Apply T Transform
  • Input: [100R, 50G, 30B]
  • Output: [63R, 60G, 44B]
Stage 2: Optional Grayscale Projection
  • If you checked the "Grayscale Projection" checkbox
  • Input: [63R, 60G, 44B]
  • Output: [50R, 50G, 50B] (all equal, so it looks gray)
Stage 3: Tone Controls (Exposure, Contrast, etc.)
  • Input: [50R, 50G, 50B]
  • Apply exposure, contrast, saturation formulas
  • Output: [52R, 48G, 49B]
Stage 4: Vibrance
  • Makes dull colors more colorful
  • Input: [52R, 48G, 49B]
  • Output: [55R, 45G, 47B]
Stage 5: Sharpen
  • Makes edges more clear
  • Compares pixel to neighbors
  • Increases difference
Stage 6: Vignette
  • Darkens edges of photo
  • Only applies to pixels near edges
Stage 7: Optional Least Squares Correction
  • If you checked "Least Squares Correction"
  • Applies an extra correction
Stage 8: Optional PCA Reduction
  • If PCA Components is not 3
  • Simplifies the colors

Key point:

Each stage is like a different filter applied on top of the previous one.

Like applying Instagram filters one after another.

PART 5: Full Backend Pipeline Order

Step by Step Processing

When you upload a photo, this happens in this order:

Step 1: Load and Resize

  • Read the image file from your computer
  • If it's too big, make it smaller (for preview speed)
  • Example: 4000 × 3000 → downscale to 1280 × 960

Step 2: Convert to Table (X)

  • Take the image and flatten it into table format
  • Each pixel becomes one row
  • If image is 1280 × 960, you have 1,228,800 rows
  • Each row has 3 columns (R, G, B)

Step 3: Build the Transform Recipe (T)

This is the most important step.

Step 3a: Get Preset Matrix T_p

  • You chose a preset like "Cinematic" or "Sepia"
  • This preset is a pre-made 3×3 matrix

Example Sepia preset:

T_p = [
  [0.393, 0.769, 0.189],
  [0.349, 0.686, 0.168],
  [0.272, 0.534, 0.131]
]

Step 3b: Build Hue Rotation R_theta

  • You moved the Hue slider to 30 degrees
  • This creates a rotation in color space
  • Think of rotating colors on a color wheel
  • Creates new matrix R_theta

Step 3c: Blend with Identity

  • You moved Filter Strength to 0.5
  • Formula: T = (1 - 0.5) × I + 0.5 × (R_theta × T_p)
  • Meaning: T = 0.5 × I + 0.5 × (rotated preset)
  • Half identity, half rotated preset
  • Result: T (final recipe matrix)

Step 4: Apply Transform to Every Pixel

  • Take recipe T
  • For each of 1.2 million pixels:
  • Multiply pixel by T
  • Store result

Steps 5-11: Post-Processing Stages

  • Step 5: Grayscale projection (if enabled)
  • Step 6: Tone controls (exposure, contrast, saturation, temperature, tint, gamma)
  • Step 7: Vibrance enhancement
  • Step 8: Sharpen edges
  • Step 9: Vignette darkening
  • Step 10: Least squares correction (if enabled)
  • Step 11: PCA reduction (if components < 3)

Step 5: Check Grayscale

  • If you checked Grayscale Projection:
  • For each pixel, project it to grayscale line
  • All pixels become [G, G, G] where G is luminance

Step 6: Apply Tone Controls

  • If you moved Exposure slider to 0.5: brighten by multiplying each value by 2^0.5
  • If you moved Contrast slider: expand values away from 0.5
  • If you moved Saturation slider: move toward or away from gray
  • Repeat for Temperature, Tint, and Gamma
  • Each one modifies the pixel values

Step 7: Apply Vibrance

  • Look at each pixel
  • Calculate how saturated it is
  • If not very saturated, boost it more
  • If already saturated, boost it less

Step 8: Apply Sharpen

  • Look at each pixel and its neighbors
  • If it's very different from neighbors, make it more different
  • Creates edge enhancement

Step 9: Apply Vignette

  • Calculate distance from pixel to center of image
  • If pixel is near edge, darken it
  • If pixel is near center, do not change it

Step 10: Check Least Squares

  • If Least Squares Correction is checked:
  • Apply extra correction matrix X_hat
  • This tries to make the image colors more balanced

Step 11: Check PCA

  • If PCA Components is less than 3:
  • Reduce to lower-dimensional color space
  • If k=2: forget one weak color direction
  • If k=1: force everything onto dominant direction

Step 12: Convert Back to Image

  • Take the table (now modified)
  • Convert back to pixel format
  • Create actual image file

Step 13: Compute Dashboard Metrics

  • Calculate Rank, Nullity, Determinant, etc.
  • These describe the T matrix
  • Show metrics to user

Step 14: Return Results

  • Send edited image to browser
  • Send all metrics
  • Browser displays everything

PART 6: Controls Explained in Depth

The 16 Essential Controls

6.1: PRESET FILTER

A pre-made color recipe that defines the base matrix T_p.

Available presets:

1. Identity:

[ [1, 0, 0],
  [0, 1, 0],
  [0, 0, 1] ]

Don't change anything. Red stays red, green stays green, blue stays blue.

2. Sepia:

[ [0.393, 0.769, 0.189],
  [0.349, 0.686, 0.168],
  [0.272, 0.534, 0.131] ]

Let's trace one pixel with Sepia: Input [100R, 50G, 30B]

  • New Red = 0.393×100 + 0.769×50 + 0.189×30 = 83.42
  • New Green = 0.349×100 + 0.686×50 + 0.168×30 = 74.24
  • New Blue = 0.272×100 + 0.534×50 + 0.131×30 = 57.83
  • Output: [83R, 74G, 58B] — Brownish tone = Sepia look!

Other presets: Cinematic, Vintage, Teal and Gold, Noir, Cool, Warm, No Red

HUGE Effect on image: Main color transformation.

6.2: FILTER STRENGTH (0 to 1)

How much of the preset to use. Blends between no change and full preset.

Formula:

T = (1 - s) × I + s × (Rotated Preset)

s = 0: No color change. s = 0.5: Half effect. s = 1: Full effect.

STRONG Controls how visible the preset is.

6.3: HUE ROTATION (-180 to 180°)

Rotates colors on a color wheel. Rotates ONLY color info, NOT brightness.

Examples:

+90°: Red→Blue, Blue→Cyan, Green→Purple

180°: Red→Cyan, Green→Magenta, Blue→Yellow

STRONG All colors shift on the wheel.

6.4: EXPOSURE (-1.5 to 1.5)

Make image brighter or darker using exponential scaling.

Formula:

new_pixel = old_pixel × 2^e

e = 1: Everything doubles. e = -1: Everything halves.

STRONG Uniform brightness adjustment.

6.5: CONTRAST (-1 to 1)

Make differences between light and dark areas bigger or smaller.

Formula:

new = clip((old - 0.5) × (1 + c) + 0.5)

Lights get lighter, darks get darker (for positive contrast).

STRONG Changes punch and drama of image.

6.6: SATURATION (-1 to 1)

How vibrant the colors are.

Formula:

  • Find luminance: g = 0.299×R + 0.587×G + 0.114×B
  • new = clip(g + (old - g) × (1 + s))

Positive values make colors more vibrant. Negative makes them grayish.

STRONG Makes colors vivid or muted.

6.7: VIBRANCE (-1 to 1)

Like saturation, but SMART. Boosts dull colors more than vibrant ones.

Dull parts get more vibrant. Already-vibrant parts stay vibrant. Natural looking.

MODERATE Smarter than saturation.

6.8: TEMPERATURE (-1 to 1)

Make image warmer (red/orange) or cooler (blue).

Formula:

R_new = clip(R + 0.08 × tau); B_new = clip(B - 0.08 × tau)

Positive: Warmer (sunset). Negative: Cooler (ice).

MODERATE Changes color temperature feeling.

6.9: TINT (-1 to 1)

Shift between green and magenta.

Formula:

G_new = clip(G + 0.06 × eta)

Positive: Greenish. Negative: Magenta.

WEAK Subtle color shift.

6.10: GAMMA (0.3 to 2.2)

Power-law remapping. Changes how dark/light values are distributed.

Formula:

new = old ^ (1/gamma)

gamma = 2: Dark areas get brighter. gamma = 0.5: Dark areas get darker.

STRONG Changes tonal distribution dramatically.

6.11: SHARPEN (0 to 2)

Unsharp mask. Makes edges crisp by enhancing differences.

Formula:

new = clip(old + alpha × (old - blur))

At edges: light becomes lighter, dark becomes darker = sharper.

MODERATE Makes edges crisper, details pop.

6.12: VIGNETTE (0 to 1)

Darken edges of photo. Like old camera or spotlight effect.

Center stays bright. Edges gradually darken. Creates dramatic framing.

MODERATE Adds dramatic edges.

6.13: PCA COMPONENTS (1 to 3)

The only control using actual Principal Component Analysis.

The Sheep Flock Analogy:

Each sheep has height, weight, wool thickness. They follow patterns: tall sheep tend to be heavy. So you only need two numbers: "Size" and "Wool". You've compressed 3 measurements to 2.

In color terms:

Each pixel has 3 colors: R, G, B. But they're not independent! PCA finds patterns and removes weak color information.

Algorithm:

  1. Normalize to 0-1
  2. Calculate mean pixel
  3. Center by subtracting mean
  4. Calculate covariance matrix
  5. Find eigenvectors (directions of most variation)
  6. Choose k components (keep top k)
  7. Reconstruct and project back

Effects:

k=3: No change. Full color. k=2: Moderate loss, muted. k=1: Extreme, almost grayscale, artistic.

VARIABLE Depends on k value.

6.14: PREVIEW SIZE (600 to max)

How big a preview to compute.

Why:

4000×3000 is slow. 1280×960 is fast. Control the computation speed.

Important:

Downloaded image ALWAYS uses original resolution. This only affects preview speed.

6.15: GRAYSCALE PROJECTION

What it is: A geometric projection in 3D color space.

Important: This is a linear algebra concept.

Core idea

RGB is a 3D space, and luminance gives a special direction:

u = [0.299, 0.587, 0.114]

When grayscale projection is enabled, each pixel is projected to the luminance line.

Formula

proj_u(v) = ((v · u) / (u · u)) × u

Simple luminance form used in practice

L = 0.299R + 0.587G + 0.114B
pixel_out = [L, L, L]

Worked example

For v = [100, 50, 30], luminance is:

L = 0.299×100 + 0.587×50 + 0.114×30 = 62.67
pixel_out = [62.67, 62.67, 62.67]

STRONG Makes image grayscale while preserving perceived brightness.

6.16: LEAST SQUARES CORRECTION

What it is: A global optimization to balance image colors.

Important: This is an optimization problem.

Optimization objective

min ||AX - B||_F^2

Closed-form solution

X_hat = (A^T A)^(-1) A^T B

How it is applied

x_corrected = x × X_hat

Constraint intuition

  • Keep red, green, and blue channels reasonably aligned with identity
  • Push global average color toward neutral gray
  • Find best compromise when exact constraints cannot all be satisfied

MODERATE Produces subtle to moderate global color balancing.

PART 7: Main Formulas

These are the live formulas shown at top of dashboard.

Transformation tile

T = (1-s)I + s R_theta T_p

This is the current blend formula.

  • s = strength slider value
  • R_theta = hue rotation matrix (theta in degrees)
  • T_p = preset matrix
  • I = identity matrix

Below this tile: rank(T) and det(T).

Projection tile

If grayscale ON:

proj_u(v) = ((v·u)/(u·u))u

If grayscale OFF:

v_out = v'

Least Squares tile

X_hat = arg min_X ||AX - B||_F^2

If LS correction is OFF: "X_hat computed only" (calculated but not used).

If LS correction is ON: "X_out = X × X_hat" (applied to pixels).

Covariance tile

C = (1/(N-1)) X_c^T X_c

Where X_c = centered pixels (X - mean).

Below this tile: lambda_1 and rho_1 (PC1 variance %).

PART 10: Summary

This app is a visual linear algebra demonstration

Step 1: Represent image as matrix X with N rows and 3 color columns.

Step 2: Apply 3x3 transform matrix T to every pixel.

Step 3: Optionally project pixels to luminance line for grayscale.

Step 4: Apply tone controls (exposure, contrast, saturation, etc.).

Step 5: Improve visual quality with vibrance, sharpen, and vignette.

Step 6: Optionally balance global color using least squares correction.

Step 7: Optionally reduce color dimensions via PCA.

Step 8: Report matrix diagnostics and statistics in dashboard tiles.

PART 11: FAQ and Quick Clarifications

Q: What does "matrix" mean in simple terms?

A: A table of numbers arranged in rows and columns.

Q: What does "eigenvalue" mean?

A: A number that describes how much variation exists in a specific direction.

Q: What does "covariance" mean?

A: How much two quantities vary together.

Q: What does "normalize" mean?

A: Scale values to a standard range, usually 0-1 or 0-255.

Q: What does "clip" mean?

A: Clamp values to a range.

Q: Why 0.299, 0.587, 0.114 for luminance?

A: Human vision is most sensitive to green, then red, then blue.

Q: Why use matrix multiplication?

A: It applies one transform to all pixels and captures channel mixing.

Q: Saturation vs Vibrance?

A: Saturation affects all colors equally; vibrance boosts dull colors more.

Q: What is Frobenius norm?

A: The square root of sum of squares of matrix elements.

Q: Why is Least Squares useful?

A: It finds the best compromise when all constraints cannot be perfectly satisfied.

Q: What happens if PCA = 1?

A: Color collapses to one dominant direction; image becomes heavily compressed.
